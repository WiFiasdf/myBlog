# 前言

在数字化时代，文件传输已经成为我们日常工作和生活中不可或缺的一部分。无论是在电脑之间传输大文件，还是在手机与电脑之间同步数据，我们都需要高效、可靠的文件传输方式。随着技术的发展，文件传输方式也日益多样化，从传统的有线传输到现代的无线传输，从简单的复制粘贴到专业的传输协议，各种方案层出不穷。

不同的传输方式各有优劣，适用于不同的场景和需求。有线传输速度快且稳定，但需要物理连接；无线传输灵活便捷，但速度和稳定性相对较低。选择合适的传输方式，可以大大提高工作效率，节省宝贵的时间。

本文将全面介绍各种文件传输方式，包括有线传输和无线传输两大类，详细讲解每种方法的原理、操作步骤、优缺点及适用场景，帮助您在不同情况下选择最合适的传输方案。

# 基础知识：文件传输的基本概念

## 什么是文件传输？

文件传输是指将文件从一个存储设备或计算机复制到另一个存储设备或计算机的过程。这个过程涉及数据的读取、传输和写入三个基本步骤。文件传输可以在同一台计算机的不同存储设备之间进行，也可以在不同的计算机之间进行，甚至可以通过网络在远程设备之间进行。

> **什么是数据流？**
> 数据流（Data Stream）是指在文件传输过程中，数据像水流一样从源设备流向目标设备的过程。数据流可以是连续的，也可以是分块的。在传输过程中，数据流会经过各种协议层，确保数据的完整性和可靠性。理解数据流的概念有助于我们理解文件传输的原理和性能瓶颈。

## 文件传输的关键指标

**1. 传输速度**：指单位时间内传输的数据量，通常以 MB/s（兆字节每秒）或 GB/s（千兆字节每秒）表示。传输速度受限于传输协议、硬件设备和网络环境等多个因素。

**2. 传输距离**：指文件传输能够跨越的物理距离。有线传输通常受限于线缆长度，无线传输则受限于信号覆盖范围。

**3. 稳定性**：指传输过程的可靠程度，包括传输中断率、数据完整性等。稳定性好的传输方式能够保证数据不丢失、不损坏。

**4. 便捷性**：指使用该传输方式的难易程度，包括设备连接、操作步骤、界面友好性等。

**5. 安全性**：指传输过程中的数据保护能力，包括加密传输、身份验证、访问控制等。

## 文件传输的基本原理

文件传输的基本原理可以分为以下几个步骤：

**1. 文件读取**：源设备从存储介质中读取文件数据，将数据加载到内存中。

**2. 数据编码**：将读取的数据按照传输协议进行编码，添加必要的控制信息和校验信息。

**3. 数据传输**：通过传输介质（如 USB 线缆、网络等）将编码后的数据发送到目标设备。

**4. 数据解码**：目标设备接收数据后，按照传输协议进行解码，提取出原始数据。

**5. 文件写入**：将解码后的数据写入目标设备的存储介质中，完成文件复制。

> **什么是校验信息？**
> 校验信息（Checksum）是用于验证数据完整性的特殊数据。在文件传输过程中，系统会为每个数据块计算校验值（如 CRC32、MD5 等），并在传输时一同发送。接收端收到数据后，重新计算校验值并与发送端的校验值进行比较，如果一致则说明数据完整，不一致则说明数据在传输过程中发生了错误或损坏。

## 常见传输协议简介

在文件传输中，不同的协议服务于不同的场景。了解它们的基本特点，有助于理解各种传输工具的工作原理。

| 协议 | 全称 | 用途 | 安全性 | 典型场景 |
| ---- | ---- | ---- | ------ | -------- |
| SMB | Server Message Block | Windows 文件共享 | 局域网内使用较安全 | 网线直连、局域网共享文件夹 |
| FTP | File Transfer Protocol | 文件上传下载 | ❗ 明文传输，不安全 | 内网文件服务器（如 WingFTP） |
| FTPS | FTP over SSL/TLS | 加密文件传输 | ✅ 加密 | 需要安全性的 FTP 场景 |
| SFTP | SSH File Transfer Protocol | 加密文件传输 | ✅ 加密 | 远程服务器文件管理 |
| HTTP/HTTPS | HyperText Transfer Protocol | 网页下载、云盘 | HTTPS 加密 | 云盘下载、网页传文件 |
| WebDAV | Web-based Distributed Authoring | 网络文件管理 | 取决于是否使用 HTTPS | 网络硬盘/NAS 文件访问 |
| P2P/局域网广播 | 点对点传输 | 局域网设备发现与传输 | 局域网内使用 | LocalSend、LANDrop、AirDrop 等 |

> **小提示**：本文介绍的各种工具底层都基于以上某种协议工作。例如，网线直连共享文件夹用的是 SMB 协议，WingFTP 用的是 FTP 协议，LocalSend 等工具则使用基于 HTTPS 的局域网发现与传输机制。

## 大文件与小文件的传输差异

你可能会发现，同样是 10 GB 的数据，复制 1 个大文件和复制 10 万个小文件，速度差异可能达数倍乃至数十倍。这是因为：

- **文件系统开销**：每传输一个文件，系统都需要创建文件条目、分配存储空间、写入元数据（文件名、时间戳、权限等），小文件数量越多，这部分开销越大
- **协议握手次数**：每个文件的传输都有“建立连接→协商参数→传输数据→确认完成”的过程，10 万个文件就要重复 10 万次
- **校验开销**：每个文件都需要单独计算和核对校验值

**实用建议**：如果需要传输大量小文件，建议先将它们压缩打包成一个文件再传输，可以大幅提高传输速度。

# 有线传输类

## USB 协议传输

### USB 协议简介

USB（Universal Serial Bus，通用串行总线）是目前最常用的外设连接和文件传输协议之一。USB 协议经历了多个版本的演进，传输速度不断提升。

> **什么是 USB 协议？**
> USB（Universal Serial Bus，通用串行总线）是一种用于连接计算机与外部设备的串行总线标准。USB 协议定义了硬件接口、传输协议、电源管理等方面的规范。USB 具有即插即用、热插拔、支持多种设备类型等特点，是目前最普及的外设连接方式。USB 协议的版本包括 USB 1.0、USB 2.0、USB 3.0（USB 3.1、USB 3.2）和 USB4 等，每个版本都带来了更高的传输速度和更强的功能。

### USB 版本与传输速度

| USB 版本 | 别名 | 理论传输速度 | 实际传输速度 | 特点 |
| --- | --- | --- | --- | --- |
| USB 1.0 | - | 12 Mbps | 约 1 MB/s | 早期版本，速度慢 |
| USB 2.0 | High Speed | 480 Mbps | 约 30-40 MB/s | 兼容性好，速度适中 |
| USB 3.0 | USB 3.1 Gen 1 | 5 Gbps | 约 400-500 MB/s | 速度快，常见蓝色接口 |
| USB 3.1 | USB 3.1 Gen 2 | 10 Gbps | 约 800-900 MB/s | 更高速，部分厂商用红色或青色接口 |
| USB 3.2 Gen 2x2 | USB 3.2 Gen 2×2 | 20 Gbps | 约 1500-1700 MB/s | 双通道，极快 |
| USB4 | - | 40 Gbps | 约 3000-3500 MB/s | 基于 Thunderbolt 3 技术，与 Thunderbolt 3 兼容但不完全相同 |

**注意**：
- 表中“实际传输速度”为典型参考值，非稳定承诺。实际速度受以下因素影响：
  - **存储设备性能**：普通 U 盘的读写速度远低于固态移动硬盘，即使接口支持高速，实际传输会被存储介质拖慢
  - **文件大小与数量**：大量小文件的传输速度会显著低于单个大文件（参见下文“大文件与小文件的传输差异”）
  - **协议栈与系统缓存**：操作系统、文件系统、数据线质量等都会影响实际速度
  - 建议以实际复制大文件（如 1 GB 以上）测试为准
- USB 3.1 Gen 1 与 USB 3.0 相同（5 Gbps），USB 3.1 Gen 2 为 10 Gbps
- USB4 是基于 Thunderbolt 3 技术的规范，但与 Thunderbolt 3 不完全相同
- 接口颜色并非官方强制标准，不同厂商可能有不同颜色，建议以接口标识为准

### USB 传输方式

#### 方式一：USB 存储设备传输

**适用场景**：需要传输大量文件，或在没有网络的环境下传输文件

**操作步骤**：

1. 准备 USB 闪存盘（U 盘）或移动硬盘
2. 将 USB 设备插入源电脑的 USB 接口
3. 等待系统识别设备，打开"此电脑"
4. 找到 USB 设备，双击打开
5. 将需要传输的文件复制或拖拽到 USB 设备中
6. 等待复制完成后，安全移除 USB 设备
7. 将 USB 设备插入目标电脑的 USB 接口
8. 打开 USB 设备，将文件复制或拖拽到目标位置

**优点**：
- 操作简单，无需额外软件
- 适用于任何有 USB 接口的设备
- 可以离线传输，不受网络限制
- 可以作为临时存储设备

**缺点**：
- 需要物理介质（U 盘或移动硬盘）
- 需要手动插拔设备，不够便捷
- 传输速度受限于 USB 版本和设备性能
- 频繁插拔可能导致接口磨损

#### 方式二：USB 数据线直连（OTG）

**适用场景**：手机与电脑之间快速传输文件

**操作步骤**：

1. 确认手机支持 OTG（On-The-Go）功能
2. 准备 USB 数据线（根据手机接口选择 Micro-USB 或 USB-C）
3. 将手机与电脑通过 USB 线连接
4. 手机上会弹出"USB 用途"选项，选择"文件传输"或"MTP"模式
5. 在电脑上打开"此电脑"，会显示手机的存储设备
6. 双击打开手机存储，像操作普通文件夹一样复制文件
7. 传输完成后，在手机上选择"仅充电"或断开连接

**优点**：
- 传输速度快（取决于 USB 版本）
- 无需网络连接
- 可以同时传输多个文件
- 可以管理手机存储

**缺点**：
- 需要数据线连接
- 需要在手机上选择正确的传输模式
- 某些手机可能需要安装驱动程序

> **什么是 OTG？**
> OTG（On-The-Go）是 USB 协议的一种扩展，允许 USB 设备（如手机、平板）在没有电脑主机的情况下直接连接其他 USB 设备（如 U 盘、键盘、鼠标等）。支持 OTG 的设备可以充当 USB 主机，为其他设备供电并控制数据传输。在文件传输场景中，OTG 功能使得手机可以直接连接 U 盘或通过 USB 线与电脑进行数据传输。

#### 方式三：USB 转网卡传输

**适用场景**：需要高速传输文件，且两台电脑都有 USB 接口

**操作步骤**：

1. 准备 USB 转网线适配器（USB 网卡）
2. 将 USB 网卡插入一台电脑的 USB 接口
3. 使用网线将两台电脑的网卡连接起来
4. 配置网络设置（参考"网线直连"部分）
5. 通过共享文件夹或文件传输软件传输文件

**优点**：
- 传输速度快（取决于网卡速度）
- 可以同时传输大量文件
- 稳定性高

**缺点**：
- 需要额外的硬件设备
- 需要配置网络设置
- 操作相对复杂

## 网线直连

### 网线直连简介

网线直连是指通过网线将两台电脑直接连接起来，建立局域网，然后通过共享文件夹或文件传输软件进行文件传输。这种方式不需要路由器或交换机，成本低廉，传输速度快。

> **什么是局域网？**
> 局域网（Local Area Network，LAN）是指在某一区域内（如家庭、办公室、学校）由多台计算机互联成的计算机网络。局域网覆盖范围通常较小（几米到几公里），传输速度快，可靠性高。网线直连建立的局域网是最简单的局域网形式，只包含两台计算机，通过网线直接连接。

### 网线类型选择

**1. 直通线（Straight-through Cable）**
- 用于不同类型设备之间的连接（如电脑连接路由器）
- 线序：两端线序相同（T568B 标准）
- 现代网卡通常支持自动翻转，直通线也可以用于电脑直连

**2. 交叉线（Crossover Cable）**
- 用于相同类型设备之间的连接（如电脑连接电脑）
- 线序：两端线序不同（一端 T568A，另一端 T568B）
- 早期电脑直连需要使用交叉线

**3. 建议**：现代网卡通常支持自动 MDI-X 功能，可以自动识别并调整线序，因此使用普通直通线即可。

> **什么是 T568A 和 T568B？**
> T568A 和 T568B 是两种常见的网线线序标准，规定了网线中 8 根线芯的排列顺序。T568B 是最常用的标准，线序为：白橙、橙、白绿、蓝、白蓝、绿、白棕、棕。T568A 的线序为：白绿、绿、白橙、蓝、白蓝、橙、白棕、棕。交叉线的一端使用 T568A，另一端使用 T568B，这样可以使发送和接收线对正确连接。

### 网线直连操作步骤

#### 步骤一：物理连接

1. 准备一根网线（建议使用 CAT5e 或 CAT6 网线）
2. 将网线的一端插入电脑 A 的网口
3. 将网线的另一端插入电脑 B 的网口
4. 观察网卡指示灯，确认连接正常（通常有指示灯亮起）

#### 步骤二：配置 IP 地址

**在电脑 A 上配置**：

1. 右键点击"网络"图标，选择"打开网络和 Internet 设置"
2. 点击"更改适配器选项"
3. 右键点击"以太网"，选择"属性"
4. 双击"Internet 协议版本 4（TCP/IPv4）"
5. 选择"使用下面的 IP 地址"
6. 输入 IP 地址：`192.168.1.1`
7. 输入子网掩码：`255.255.255.0`
8. 网关和 DNS 服务器留空
9. 点击"确定"

**在电脑 B 上配置**：

1. 按照相同步骤打开网络属性
2. 输入 IP 地址：`192.168.1.2`（注意与电脑 A 的 IP 不同）
3. 输入子网掩码：`255.255.255.0`
4. 网关和 DNS 服务器留空
5. 点击"确定"

**提示**：`192.168.1.1` 与 `192.168.1.2` 为私有 IP 地址，适合局域网直连使用。

#### 步骤三：测试连接

1. 在电脑 A 上，按 `Win + R`，输入 `cmd`，回车
2. 输入命令：`ping 192.168.1.2`
3. 如果看到类似"来自 192.168.1.2 的回复"的字样，说明连接成功
4. 如果显示"请求超时"，检查 IP 地址配置和网线连接

#### 步骤四：传输文件

**方法一：通过共享文件夹传输**

1. 在电脑 A 上，右键点击要共享的文件夹
2. 选择"属性" → "共享"选项卡
3. 点击"共享"按钮
4. 在下拉菜单中选择"Everyone"或"Guest"
5. 点击"添加"，然后点击"共享"
6. 记下共享路径（如 `\\192.168.1.1\共享文件夹`）
7. 在电脑 B 上，按 `Win + R`，输入共享路径，回车
8. 现在可以像操作本地文件夹一样复制文件了

**方法二：使用文件传输软件**

参考下文"无线传输类"中的各种文件传输软件，它们大多支持局域网传输。

### 网线直连的优缺点

**优点**：
- 传输速度快（取决于网卡速度，千兆网卡典型速度可达 100-120 MB/s）
- 稳定性高，不易受干扰
- 成本低廉，只需一根网线
- 可以同时传输大量文件
- 不受互联网带宽限制

**缺点**：
- 需要物理连接，不够灵活
- 需要配置网络设置
- 两台电脑距离不能太远（网线长度限制，通常不超过 100 米）
- 操作相对复杂

### 网线直连常见问题排查

网线直连在实际操作中，很多人会卡在“能 ping 通但访问不了共享”的环节。以下是常见问题及解决方法：

**1. ping 通但访问不了共享文件夹**

- **检查网络类型**：确保两台电脑的网络配置文件为“专用网络”而非“公用网络”。公用网络默认会阻止文件共享。打开“设置”→“网络和 Internet”→“以太网”，查看网络配置文件类型，必要时切换为“专用”
- **开启网络发现和文件共享**：打开“控制面板”→“网络和共享中心”→“更改高级共享设置”，确保“启用网络发现”和“启用文件和打印机共享”已勾选
- **检查防火墙**：Windows 防火墙可能会阻止 SMB 协议（端口 445）。在 Windows 防火墙设置中，确保“文件和打印机共享”规则已启用，或临时关闭防火墙进行测试

**2. 共享路径打不开**

- **确认共享路径格式**：使用 `\\IP地址\共享名` 格式（如 `\\192.168.1.1\我的共享`），注意是反斜杠 `\` 而非正斜杠 `/`
- **检查用户名密码**：部分 Windows 版本默认不允许匿名访问，可能需要输入对方电脑的用户名和密码。如果对方电脑没有设置密码，可能需要先设置一个密码
- **尝试映射网络驱动器**：在“此电脑”中点击“映射网络驱动器”，输入共享路径，这样可以像访问本地磁盘一样访问共享文件夹

**3. 共享权限问题**

- **共享权限 vs NTFS 权限**：要同时检查“共享”选项卡和“安全”选项卡的权限设置。两者取交集（更严格的那个生效），如果共享权限允许但 NTFS 权限拒绝，仍然无法访问
- **建议**：对于临时传输，可以在共享时选择“Everyone”并赋予“读取/写入”权限，传输完成后及时取消共享

**4. 其他常见问题**

- **SMB 1.0 已禁用**：Windows 10/11 默认可能禁用了 SMB 1.0 协议。如果访问较旧的设备，可能需要在“启用或关闭 Windows 功能”中启用 SMB 1.0
- **IP 地址冲突**：确保两台电脑的 IP 地址不同但在同一网段（如 `192.168.1.x`），子网掩码相同
- **网线问题**：更换网线或检查水晶头压接是否正常

## 选择无线传输工具的参考标准

无线传输工具种类繁多，在选择具体工具前，建议从以下维度衡量：

| 考量维度 | 说明 |
| -------- | ---- |
| 是否开源 | 开源软件代码可审计，透明度和可信度更高 |
| 是否需要登录/注册 | 不需要账号的工具更保护隐私，操作也更简便 |
| 数据是否经过公网服务器 | 纯局域网传输更安全、更快；经过公网中转则受带宽和服务商策略限制 |
| 是否支持断点续传 | 传输大文件时很重要，避免中断后重新传输 |
| 是否支持文件夹整体传输 | 有些工具只能传单个文件，不支持文件夹 |
| 跨平台支持 | 确认工具是否支持你所有设备的操作系统 |
| 权限控制 | 多用户场景下，是否支持设置不同的读写权限 |

### 云盘传输简介

云盘传输是指将文件上传到云存储服务，然后在另一台设备上下载的方式。这种方式不需要设备之间直接连接，只要有网络即可。云盘服务通常提供网页版、客户端和手机 APP，方便用户在不同设备上访问。

> **什么是云存储？**
> 云存储（Cloud Storage）是一种将数据存储在云端服务器上的服务模式。用户可以通过网络随时随地访问存储在云端的文件。云存储服务提供商负责维护服务器、数据备份和安全防护，用户只需按照使用量或存储容量付费。常见的云存储服务包括百度网盘、阿里云盘、腾讯微云、OneDrive、Google Drive、Dropbox 等。

## 使用 WingFTP

### WingFTP 简介

WingFTP 是一款专业的 FTP 服务器软件，可以在电脑上搭建 FTP 服务器，然后通过 FTP 客户端从其他设备访问和下载文件。提供免费版本，但功能有限。FTP（File Transfer Protocol，文件传输协议）是一种用于在网络上进行文件传输的标准协议。

> **什么是 FTP？**
> FTP（File Transfer Protocol，文件传输协议）是用于在网络上进行文件传输的标准协议。FTP 使用客户端/服务器模式，用户通过 FTP 客户端连接到 FTP 服务器，然后可以上传、下载、删除、重命名文件等。FTP 协议支持匿名访问和身份验证，可以设置不同的访问权限。FTP 传输速度快，适合大文件传输，但明文传输不够安全，建议使用 FTPS 或 SFTP。

### WingFTP 操作步骤

#### 步骤一：安装和配置 WingFTP Server

1. 下载 WingFTP Server（https://www.wftpserver.com/）
2. 安装并运行 WingFTP Server
3. 点击"新建域"，配置域名（如 myftp）
4. 点击"新建用户"，创建用户账号和密码
5. 设置用户的根目录和权限（读取、写入、删除等）
6. 启动服务器

#### 步骤二：从其他设备访问 FTP 服务器

**通过浏览器访问**：

1. 打开浏览器
2. 在地址栏输入：`ftp://服务器 IP 地址`（如 `ftp://192.168.1.1`）
3. 输入用户名和密码
4. 现在可以像操作本地文件夹一样上传和下载文件

**通过 FTP 客户端访问**：

1. 下载并安装 FTP 客户端（如 FileZilla、WinSCP）
2. 输入服务器地址、用户名和密码
3. 连接服务器
4. 上传或下载文件

### WingFTP 的优缺点

**优点**：
- 专业的 FTP 服务器，功能强大
- 支持多用户，可以设置不同权限
- 传输速度快
- 支持断点续传
- 可以搭建私有文件服务器

**缺点**：
- 配置相对复杂
- 需要安装服务器软件
- 需要网络知识
- 免费版本功能有限
- 不适合普通用户

## 使用蓝牙（速度较慢，适合应急）

### 蓝牙传输简介

蓝牙是一种短距离无线通信技术，可以用于文件传输。蓝牙传输速度较慢，不适合大文件或追求效率的场景，但在没有网络、没有数据线的情况下，蓝牙可能是为数不多的可用选择。

> **什么是蓝牙？**
> 蓝牙（Bluetooth）是一种短距离无线通信技术，工作在 2.4 GHz 频段。蓝牙技术最初由爱立信公司开发，现在由蓝牙技术联盟（SIG）管理。蓝牙设备可以在 10 米到 100 米的范围内进行通信，支持数据传输、语音通话等多种应用。蓝牙技术经历了多个版本的演进，传输速度不断提升，但与 Wi-Fi 相比仍然较慢。

### 蓝牙传输操作步骤

#### 步骤一：配对设备

1. 在两台设备上打开蓝牙功能
2. 在一台设备上点击"添加蓝牙设备"
3. 选择另一台设备进行配对
4. 确认配对码（如果需要）
5. 等待配对成功

#### 步骤二：传输文件

**电脑与电脑之间**：

1. 右键点击要发送的文件
2. 选择"发送到" → "蓝牙设备"
3. 选择接收设备
4. 在接收设备上确认接收

**电脑与手机之间**：

1. 在电脑上右键点击文件，选择"发送到" → "蓝牙设备"
2. 选择手机
3. 手机上会提示接收文件，点击确认
4. 文件会保存到手机的蓝牙接收文件夹

**手机与手机之间**：

1. 在手机上打开文件管理器
2. 找到要发送的文件
3. 点击"分享"，选择"蓝牙"
4. 选择接收设备
5. 在接收手机上确认接收

### 蓝牙传输的优缺点

**优点**：
- 不需要网络连接
- 不需要数据线
- 设备普及率高
- 操作简单

**缺点**：
- 传输速度慢（通常在 200 KB/s - 5 MB/s 之间，视蓝牙版本与实现而定）
- 传输距离短（通常不超过 10 米）
- 不适合大文件传输
- 配对过程可能较复杂
- 传输不稳定，容易中断

**参考速度范围**：
- 蓝牙 2.0 + EDR：约 2-3 Mbps
- 蓝牙 3.0 + HS：约 24 Mbps
- 蓝牙 4.0/5.0：约 2-5 Mbps（实际速度通常低于理论速度）

3. 选择要连接的设备

#### 步骤三：传输文件

1. 选择要发送的文件或文件夹
2. 点击"发送"
3. 等待传输完成

### 互传联盟的优缺点

**优点**：
- 不需要安装额外软件
- 跨品牌支持
- 传输速度快
- 操作简单

**缺点**：
- 只支持特定品牌的手机
- 需要两台手机都支持互传联盟
- 电脑不支持

# 其他传输方式

## AirDrop（苹果设备专用）

### AirDrop 简介

AirDrop 是苹果公司推出的文件传输功能，支持 iPhone、iPad、Mac 之间的文件传输。AirDrop 使用蓝牙和 Wi-Fi 技术，传输速度快且安全。

### AirDrop 操作步骤

1. 在两台苹果设备上打开 AirDrop 功能
2. 在发送设备上选择要发送的文件
3. 点击"分享"，选择 AirDrop
4. 选择接收设备
5. 在接收设备上确认接收

### AirDrop 的优缺点

**优点**：
- 苹果生态内体验极佳
- 传输速度快
- 操作简单
- 安全性高

**缺点**：
- 只支持苹果设备
- 需要较新的设备和系统版本

---

## NFC 近场通信

### NFC 简介

NFC（Near Field Communication，近场通信）是一种短距离无线通信技术，可以用于文件传输。NFC 通常用于 Android 设备之间的文件传输，需要两台设备靠近（几厘米以内）。

### NFC 传输操作步骤

1. 在两台设备上开启 NFC 功能
2. 在发送设备上选择要发送的文件
3. 点击"分享"，选择"NFC"（如设备支持）
4. 将两台设备背靠背靠近（几厘米以内）
5. 等待传输完成

> **注意**：Google 的 Android Beam 已在 Android 10 及后续版本中移除。现代 Android 手机的文件传输通常使用厂商互传功能或 Nearby Share（快速分享），NFC 更多用于设备间的快速配对或触碰交互。

### NFC 传输的优缺点

**优点**：
- 操作简单，靠近即可
- 不需要网络连接
- 安全性高

**缺点**：
- 传输速度慢
- 传输距离极短
- 只适合小文件传输
- 需要设备支持 NFC

| 传输方式 | 传输速度 | 便捷性 | 稳定性 | 成本 | 适用场景 |
| -------- | -------- | ------ | ------ | ---- | -------- |
| 云盘传输 | 1-50 MB/s | 高 | 中等 | 免费-付费 | 远程传输、文件备份 |
| 360 极连快传 | 100+ MB/s | 高 | 高 | 免费 | 局域网传输 |

# 选择建议

## 快速决策树

不知道怎么选？按照以下流程快速判断：

```
① 两台设备在同一个局域网吗？
   ├─ 是 → ② 文件大小超过 1 GB 吗？
   │       ├─ 是 → 网线直连 / USB 3.0+ U盘 / LocalSend
   │       └─ 否 → LocalSend / LANDrop / 厂商互传
   └─ 否 → ③ 能联网吗？
           ├─ 是 → ④ 文件敏感吗？
           │       ├─ 是 → 加密压缩后云盘 / SFTP / 端到端加密工具
           │       └─ 否 → 云盘 / QQ 闪传 / 微信文件传输
           └─ 否 → ⑤ 有数据线 / U盘吗？
                   ├─ 是 → USB 数据线直连 / U盘拷贝
                   └─ 否 → 蓝牙（仅限小文件）
```

> **苹果设备用户**：苹果设备之间优先使用 AirDrop，简单快速。

## 根据传输距离选择

**同房间或近距离**：
- 优先考虑有线传输（USB、网线）
- 局域网无线传输（360 极连快传、LocalSend、LANDrop）

**跨房间或中等距离**：
- 局域网无线传输
- 云盘传输

**远程传输**：
- 云盘传输
- QQ 闪传

## 根据文件大小选择

**小文件（< 100 MB）**：
- 任何传输方式都可以
- 蓝牙、NFC 等慢速方式也可以

**中等文件（100 MB - 1 GB）**：
- USB 传输
- 局域网无线传输
- 云盘传输（可能需要等待）

**大文件（> 1 GB）**：
- 优先考虑高速传输方式
- USB 3.0+、网线直连
- 局域网无线传输（360 极连快传、LocalSend）

## 根据设备类型选择

**电脑与电脑**：
- 网线直连（速度快且稳定）
- USB 存储设备
- 局域网无线传输
- 云盘传输

**电脑与手机**：
- USB 数据线直连（速度快且稳定）
- 手机配套软件
- 局域网无线传输
- QQ 闪传
- 云盘传输

**手机与手机**：
- 互传联盟（如果支持）
- 蓝牙（小文件）
- 局域网无线传输
- QQ 闪传
- 云盘传输

**苹果设备**：
- AirDrop（最佳选择）
- 云盘传输

## 根据网络环境选择

**有网络**：
- 局域网无线传输
- 云盘传输
- QQ 闪传

**无网络**：
- 有线传输（USB、网线）
- 蓝牙
- NFC

# 常见传输失败原因与解决方法

文件传输过程中，经常会遇到各种问题。以下是最常见的失败场景及对应的解决思路：

## 搜不到设备

- **局域网工具（LocalSend、LANDrop 等）**：确认两台设备在同一个子网（同一个路由器，而非一个连有线一个连 Wi-Fi 且被路由器隔离）；检查防火墙是否阻止了该软件的网络访问；部分路由器开启了“AP 隔离”（客户端隔离）功能，会阻止设备间通信，需要在路由器设置中关闭
- **蓝牙**：确认两台设备都已开启蓝牙且设置为“可被发现”；检查距离是否过远；尝试删除已有配对后重新配对

## 传输速度异常缓慢

- **USB 传输慢**：检查是否插入了 USB 2.0 接口（通常为黑色）而非 USB 3.0 接口（通常为蓝色）；检查 U 盘/移动硬盘本身是否支持高速；大量小文件建议先压缩再传
- **局域网传输慢**：检查 Wi-Fi 是否连接在 2.4 GHz 频段（较慢）而非 5 GHz 频段；检查路由器负载；可以尝试用网线直连代替 Wi-Fi
- **云盘下载慢**：检查是否为免费用户被限速；尝试更换下载时间段；考虑使用客户端而非网页版

## 传输中断 / 传到一半断开

- **检查连接稳定性**：USB 线松动、网线接触不良、Wi-Fi 信号弱都可能导致中断
- **使用支持断点续传的工具**：如 FTP 客户端（FileZilla）、部分云盘客户端等支持断点续传，中断后可从上次位置继续
- **避免传输时操作设备**：传输过程中不要拔插设备、不要让电脑休眠

## 文件损坏 / 打不开

- **核对文件大小**：对比源文件和目标文件的大小，如果不一致，说明传输不完整
- **校验哈希值**：使用前文提到的校验命令（参见“文件完整性校验操作指南”）比对哈希值
- **文件系统限制**：FAT32 格式的 U 盘不支持单个超过 4 GB 的文件，如需传输大文件，请将 U 盘格式化为 NTFS 或 exFAT

# 注意事项

## 数据安全

1. **使用加密传输**：对于敏感文件，使用支持加密的传输方式（如 SFTP、HTTPS），避免使用明文 FTP
2. **避免使用公共 Wi-Fi 传输敏感文件**：公共网络存在被窫听的风险，如必须使用，请确保传输通道已加密
3. **定期备份**：重要文件传输后，及时备份到多个位置
4. **验证文件完整性**：传输大文件后，验证文件的完整性（如使用 MD5、SHA-256 校验）

### 文件完整性校验操作指南

传输大文件（特别是系统镜像、压缩包等）后，建议核对文件哈希值，确保文件在传输过程中未损坏。

**Windows（PowerShell）**：
```powershell
# 计算文件的 MD5 哈希值
Get-FileHash -Path "C:\你的文件路径\file.zip" -Algorithm MD5

# 计算文件的 SHA-256 哈希值（更安全，推荐）
Get-FileHash -Path "C:\你的文件路径\file.zip" -Algorithm SHA256
```

**macOS / Linux（终端）**：
```bash
# MD5 校验
md5sum /path/to/file.zip        # Linux
md5 /path/to/file.zip            # macOS

# SHA-256 校验
sha256sum /path/to/file.zip      # Linux
shasum -a 256 /path/to/file.zip  # macOS
```

将计算出的哈希值与发送方提供的值进行比对，一致则说明文件完整。

### 敏感文件传输安全清单

- ✅ 优先使用局域网直传（如 LocalSend、网线直连），避免数据经过公网
- ✅ 必须走公网时，使用 SFTP、HTTPS 或端到端加密工具
- ✅ 不要使用明文 FTP 传输敏感文件（密码和文件内容会被明文传输）
- ✅ 传输前可以先将文件加密压缩（如使用 7-Zip 设置 AES-256 加密）
- ✅ 传输完成后，核对哈希值确认文件完整
- ✅ 传输完成后，及时删除中转服务器上的临时文件（如云盘、QQ 离线文件等）

## 传输效率

1. **批量传输**：尽量一次传输多个文件，减少传输次数
2. **压缩文件**：大文件或多个小文件可以先压缩，减少传输时间
3. **选择合适的时间**：在网络不繁忙时传输大文件
4. **使用高速设备**：使用 USB 3.0+、高速网卡等设备提高传输速度

## 设备兼容性

1. **检查设备接口**：确认设备的接口类型（USB、网口等）
2. **检查协议支持**：确认设备支持的传输协议（如 USB 3.0、Wi-Fi 6）
3. **测试连接**：传输前先测试设备连接是否正常
